# Issue 003: SOPD sector profiles and workflows

## Goal
Make the repository usable by SOPDs so they can generate maps by sector, duties, and functions without redesigning the pipeline.

## Scope
- Introduce sector profiles for common SOPD themes.
- Provide default layers, titles, and export naming per sector.
- Keep the workflow consistent with BIG standards and base map conventions.
- Document the expected folder structure for each project.

## Acceptance Criteria
- A project can declare a sector profile and get a suitable map template.
- The same generator supports multiple sectors without code duplication.
- Documentation explains how SOPDs should structure their project folders.
